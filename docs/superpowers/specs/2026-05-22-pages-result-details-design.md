# Pages Result Details Design

## Goal

GitHub Pages 화면에서 저장된 구매번호만 보는 데 그치지 않고, 최근 검사에서 확인된 당첨번호와 회차별 당첨 요약을 함께 확인할 수 있게 한다.

## Scope

- `data/result-history.yml`에 회차별 공개 당첨번호와 티켓별 결과 요약을 저장한다.
- 구매번호 원문 전체를 이력 파일에 중복 저장하지 않는다.
- Pages는 기존 `지난 당첨결과 이력` 영역을 더 읽기 쉬운 결과 카드로 표시한다.
- 기존 카카오 알림, 구매번호 정리, 워크플로 실행 방식은 유지한다.

## Data Model

`write_result_history`는 기존 요약 필드를 유지하면서 `winning`과 `tickets`를 추가한다.

- 로또 `winning`에는 당첨번호 6개와 보너스 번호를 저장한다.
- 연금복권 `winning`에는 조, 당첨번호, 보너스 번호를 저장한다.
- `tickets`에는 라벨, 당첨 여부, 결과 라벨, 로또 일치 개수만 저장한다.

## UI

Pages의 이력 영역은 각 회차를 하나의 행보다 풍부한 카드로 렌더링한다.

- 첫 줄은 기존 요약을 유지한다.
- 둘째 줄은 공개 당첨번호를 표시한다.
- 아래에는 티켓별 결과 배지를 표시한다.

## Testing

- Python unittest로 이력 파일에 당첨번호와 티켓별 결과가 저장되는지 검증한다.
- 같은 테스트에서 구매번호 전체가 이력 파일에 저장되지 않는지 검증한다.
- Pages 스크립트는 브라우저 문법 검증과 전체 테스트로 확인한다.
