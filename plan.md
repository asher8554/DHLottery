# 계획

## 목표

동행복권 구매 자동화와 은행이체 자동화는 구현하지 않는다.
대신 사용자가 직접 구매한 로또와 연금복권 번호를 안전하게 기록하고, 발표 후 일정 여유 시간이 지난 뒤 당첨 결과를 확인해 카카오톡으로 알림을 보낸다.

## 범위

- 로또 6/45 당첨번호 조회.
- 연금복권720+ 당첨번호 조회.
- 사용자가 입력한 구매 번호와 당첨번호 비교.
- 당첨 등수와 당첨금 계산 또는 표시.
- GitHub Actions 스케줄 실행.
- Kakao Developers의 나에게 보내기 API를 통한 알림.
- 민감한 값은 GitHub Actions Secrets 또는 로컬 `.env`로만 주입.

## 제외 범위

- 동행복권 로그인 자동화.
- 복권 자동 구매.
- 예치금 충전 자동화.
- 은행이체 자동화.
- 사용자의 동행복권 계정 정보 저장.

## 성공 기준

- 테스트 데이터로 로또와 연금복권 등수 계산을 검증한다.
- GitHub Actions에서 실행 가능한 명령을 제공한다.
- 구매번호 예시 파일은 저장소에 올리되, 실제 구매번호 파일은 기본적으로 무시한다.
- 카카오 토큰과 같은 비밀값은 저장소에 커밋되지 않도록 한다.
- 사용자가 GitHub와 Kakao에서 직접 해야 할 작업을 문서화한다.

## 2026-05-10 티켓 붙여넣기 입력 도구

목표는 동행복권 로그인이나 구매 자동화 없이, 사용자가 구매 후 티켓 보기 또는 구매내역 화면의 텍스트를 복사해 붙여넣으면 실제 선택번호만 `data/tickets.yml`에 저장하는 것이다.

1. 붙여넣은 텍스트에서 로또6/45 회차와 게임별 선택번호를 찾는다.
   검증은 예시 티켓 텍스트로 파서 단위 테스트를 작성한다.
2. 티켓 식별번호처럼 45보다 큰 숫자 묶음은 선택번호로 저장하지 않는다.
   검증은 `64315 53510 57779 62575 50796 26908` 입력이 거부되는 테스트로 확인한다.
3. CLI 명령으로 `data/tickets.yml`을 갱신한다.
   검증은 임시 파일을 사용해 YAML 출력과 기존 설정 로딩을 확인한다.
4. 비밀번호, 로그인 세션, 예치금 충전 정보는 읽거나 저장하지 않는다.
   검증은 `.env`나 계정 정보 관련 코드를 추가하지 않는 것으로 확인한다.

## 2026-05-10 간단 번호 입력 형식

목표는 `1224회` 다음에 로또 숫자 6개만 줄 단위로 붙여넣어도 기존 `import-ticket` 명령으로 저장되게 하는 것이다.

1. `로또6/45` 문구가 없는 간단 형식에서도 회차와 번호 6개를 읽는다.
   검증은 사용자가 제시한 줄바꿈 형식을 단위 테스트로 확인한다.
2. 날짜, 티켓 식별번호처럼 45보다 큰 숫자가 섞인 텍스트는 간단 형식으로 받아들이지 않는다.
   검증은 기존 식별번호 거부 테스트를 유지한다.

## 2026-05-10 GitHub Pages 입력 화면 예시

목표는 GitHub Pages로 올릴 수 있는 정적 페이지에서 회차와 로또 번호 6개를 입력하면 `tickets.yml` 내용을 생성해 사용자가 바로 복사하거나 내려받을 수 있게 하는 것이다.

1. `docs/ticket-entry.html`에 정적 입력 화면을 만든다.
   검증은 브라우저 없이 HTML 파일 존재와 핵심 입력 요소를 확인한다.
2. 회차, 번호 범위, 중복 번호를 브라우저에서 검증한다.
   검증은 페이지 스크립트에 1부터 45 사이 중복 없는 숫자 조건을 둔다.
3. GitHub Pages가 저장소 파일이나 Secret을 직접 수정하지 않는 한계를 문서에 명확히 둔다.
   검증은 README에 예시 페이지 링크와 사용 범위를 추가한다.
