# 계획

## 목표

동행복권 구매 자동화와 은행이체 자동화는 구현하지 않는다.
대신 사용자가 직접 구매한 로또와 연금복권 번호를 안전하게 기록하고, 발표 후 일정 여유 시간이 지난 뒤 당첨 결과를 확인해 카카오톡으로 알림을 보낸다.

## 범위

- 로또 6/45 당첨번호 조회.
- 연금복권720+ 당첨번호 조회.
- 사용자가 입력한 구매 번호와 당첨번호 비교.
- 당첨 등수와 당첨금 계산 또는 표시.
- GitHub Actions 스케줄 실행.
- Kakao Developers의 나에게 보내기 API를 통한 알림.
- 민감한 값은 GitHub Actions Secrets 또는 로컬 `.env`로만 주입.

## 제외 범위

- 동행복권 로그인 자동화.
- 복권 자동 구매.
- 예치금 충전 자동화.
- 은행이체 자동화.
- 사용자의 동행복권 계정 정보 저장.

## 성공 기준

- 테스트 데이터로 로또와 연금복권 등수 계산을 검증한다.
- GitHub Actions에서 실행 가능한 명령을 제공한다.
- 구매번호 예시 파일은 저장소에 올리되, 실제 구매번호 파일은 기본적으로 무시한다.
- 카카오 토큰과 같은 비밀값은 저장소에 커밋되지 않도록 한다.
- 사용자가 GitHub와 Kakao에서 직접 해야 할 작업을 문서화한다.

## 2026-05-10 공개 티켓 파일 갱신 자동화

1. `data/tickets.yml`을 공개 커밋 대상으로 바꾼다.
   검증은 `.gitignore`에서 해당 경로를 제거하고 현재 파일을 추적 대상으로 올리는 것으로 한다.

   검증은 workflow_dispatch 입력, `contents: write`, 파일 검증, 커밋, 검사 실행 단계가 있는지 확인한다.
3. 기존 스케줄 검사 워크플로가 공개 파일을 읽게 바꾼다.
   검증은 `check-results.yml`이 `--tickets data/tickets.yml`을 사용하고 `TICKETS_YAML` Secret에 의존하지 않는지 확인한다.
4. 웹페이지에서 GitHub token으로 workflow_dispatch를 호출한다.
   검증은 정적 HTML에 API URL, token 입력, 저장 버튼, Actions 링크가 있는지 확인한다.

## 2026-05-10 1분 초과 조회를 발표 전으로 처리
